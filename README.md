# AI-like Score vs Academic Writing Style

This repository contains scripts and experiment notes for an exploratory study on whether AI-detection scores measure AI authorship or academic writing style.

The project builds a controlled abstract-level dataset with three text types:

- `published_abstract`: real published abstracts from PubMed/arXiv metadata.
- `ai_generate_from_metadata`: abstracts generated by LLMs from title/year/source metadata.
- `ai_polish_original_abstract`: original human abstracts polished by LLMs without changing the intended meaning.

The core claim is deliberately narrow: an AI-like score should not be interpreted as a direct measure of real AI participation. In this experiment, generated abstracts are detected much more strongly than AI-polished human-semantic abstracts.

## Main Results

Final merged dataset:

| Text type | Count |
|---|---:|
| Real published abstracts | 1,669 |
| AI-generated abstracts from metadata | 1,112 |
| AI-polished original abstracts | 1,109 |
| Total texts | 3,890 |

Grouped holdout evaluation uses `source_id` splitting, so different variants of the same source paper cannot appear in both training and test sets.

| Model | Accuracy | Precision | Recall | F1 | ROC-AUC |
|---|---:|---:|---:|---:|---:|
| Text TF-IDF logistic regression | 0.948 | 0.909 | 0.963 | 0.935 | 0.987 |
| Style-feature logistic regression | 0.805 | 0.709 | 0.856 | 0.775 | 0.885 |
| Text + style logistic regression | 0.939 | 0.883 | 0.974 | 0.926 | 0.987 |

Combined-model score distribution:

| Group | Mean AI-like score | Share >= 0.5 |
|---|---:|---:|
| AI-generated abstracts | 0.865 | 97.8% |
| AI-polished abstracts | 0.362 | 31.9% |
| post-ChatGPT real abstracts | 0.157 | 7.2% |
| pre-ChatGPT real abstracts | 0.127 | 3.0% |

## Repository Layout

Key scripts:

- `collect_multisource_abstracts.py`: collect PubMed/arXiv/OpenAlex-style metadata and abstracts.
- `merge_multisource_real_abstracts.py`: merge and deduplicate real abstract sources.
- `make_ai_prompts_multimodel.py`: build per-row LLM prompt tasks.
- `call_siliconflow_controls_multimodel.py`: call SiliconFlow-compatible chat completion APIs using per-row model assignments.
- `build_ai_control_dataset.py`: merge real abstracts with valid AI-control outputs.
- `relabel_dataset_v2.py`: apply the three-way text-origin label scheme.
- `extract_style_features.py`: compute transparent style features.
- `train_local_ai_detector.py`: train and evaluate grouped local AI-like detectors.
- `plot_detector_distributions.py`: generate Chinese distribution plots.
- `analyze_detector_cases.py`: inspect high/low scoring examples.

Generated datasets, model binaries, temporary files, and API outputs are intentionally excluded from Git by `.gitignore`.

## API Keys

Do not hard-code API keys in any script.

Use environment variables instead:

```powershell
$env:SILICONFLOW_API_KEY="YOUR_KEY"
$env:NCBI_API_KEY="YOUR_KEY"
$env:OPENALEX_API_KEY="YOUR_KEY"
```

The scripts read keys with `os.getenv(...)`. Before publishing, scan for accidental secrets:

```powershell
Select-String -Path .\*.py,.\*.md -Pattern "api_key\s*=|SILICONFLOW_API_KEY|NCBI_API_KEY|OPENALEX_API_KEY|Bearer "
```

## Minimal Workflow

Collect or prepare real abstracts:

```powershell
python .\collect_multisource_abstracts.py `
  --sources pubmed,arxiv `
  --years 2010-2026 `
  --per-year 100 `
  --email you@example.com `
  --out real_abstracts.csv `
  --use-certifi
```

Generate AI-control prompt tasks:

```powershell
python .\make_ai_prompts_multimodel.py `
  --input .\real_abstracts.csv `
  --out .\ai_control_prompts.csv `
  --models "deepseek-ai/DeepSeek-V4-Pro,tencent/Hunyuan-MT-7B" `
  --assignment round_robin
```

Call the LLM API:

```powershell
python .\call_siliconflow_controls_multimodel.py `
  --input .\ai_control_prompts.csv `
  --out .\ai_controls_generated.csv `
  --base-url "https://api.siliconflow.cn/v1" `
  --workers 8 `
  --save-every 20
```

Build the modeling dataset:

```powershell
python .\build_ai_control_dataset.py `
  --real .\real_abstracts.csv `
  --generated .\ai_controls_generated.csv `
  --out .\all_abstract_variants.csv `
  --allow-errors

python .\relabel_dataset_v2.py `
  --input .\all_abstract_variants.csv `
  --out .\all_abstract_variants_v2.csv

python .\extract_style_features.py `
  --input .\all_abstract_variants_v2.csv `
  --out .\all_abstract_variants_style_features_v2.csv
```

Train grouped detectors:

```powershell
python .\train_local_ai_detector.py `
  --input .\all_abstract_variants_style_features_v2.csv `
  --out-dir .\local_detector_results_grouped_v2
```

Plot distributions:

```powershell
python .\plot_detector_distributions.py `
  --input .\local_detector_results_grouped_v2\predictions.csv `
  --out-dir .\local_detector_results_grouped_v2\figures_cn
```

## Label Interpretation

The project distinguishes semantic authorship from surface editing:

- `human_original`: real published abstracts.
- `ai_generated`: LLM-generated abstracts from metadata, used as positive examples for strict AI-generation detection.
- `human_semantics_ai_polished`: original human semantics with AI surface polishing, not treated as strict AI-generated text.

This distinction matters because AI-polished human text can receive much lower AI-like scores than fully AI-generated text.

## Limitations

- The local detector is not equivalent to commercial AI detectors.
- The current data are primarily PubMed/arXiv abstracts.
- LLM-generated controls depend on prompt design and model availability.
- AI-like scores are classifier scores under a specific training definition, not measured percentages of AI involvement.
